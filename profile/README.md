## 为什么安全性是选 VPS 的第一道门槛

有些人觉得安全是大企业才需要操心的事，个人用户或者小站长无所谓。这个想法其实存在几个误区值得先说清楚。

**数据丢失的风险被严重低估。** 没有任何备份机制的 VPS，一旦节点故障，所有内容都可能不翼而飞。对于跑着个人博客、代理或者自建服务的用户，这意味着从头再来。

**IP 被封锁的代价很高。** 国内用户使用海外 VPS 时，IP 被封是常见风险。如果服务商没有提供机房迁移或更换 IP 的途径，一台机器就直接废掉了，钱和时间全部白费。

**网络中断没有预警机制。** 一些低价 VPS 商家对网络稳定性的投入极少，故障发生后几天都没有反应，用户只能眼睁睁等着。

选 VPS 之前，先搞清楚这台机器在这三件事上到底有没有保障，比对比 CPU 型号要重要得多。

## 安全配置的最佳实践建议

买到手之后，还有几件事值得做，能让安全保障落地更彻底：

**立即修改 SSH 默认端口。** KiwiVM 控制面板提供了 SSH 默认端口信息，建议在系统配置好之后将其修改为非标准端口，减少暴力破解风险。

**开启 Root 以外的管理账户。** 不建议长期用 Root 账户直接操作，建立权限受限的管理账户是基本安全习惯。

**启用防火墙规则。** 搬瓦工提供的是 Self-managed VPS，防火墙配置完全由用户自行决定。iptables 或 ufw 的基础规则，建议装完系统第一时间配置好。

**定期手动创建快照。** 虽然搬瓦工提供自动备份，手动快照可以在重要操作（比如升级系统或部署新应用）之前主动做一个"存档点"，出问题能直接回滚。

**善用机房迁移应对线路问题。** 如果发现某个机房丢包率明显上升或速度变慢，不要硬撑，直接在 KiwiVM 后台迁到备选机房，整个过程数据保留，通常十几分钟内完成。
